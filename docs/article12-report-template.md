# Article 12 Audit Report Template
## EU AI Act — High-Risk AI System Compliance Report

This document defines the structure and required content of the PDF audit
report generated by the Ochain compliance layer. Each section maps to a
specific Article 12 obligation. The report is generated programmatically;
this template is the canonical reference for the report builder.

---

## Report Structure

```
1. Cover page + integrity verification block (QR code + report fingerprint)
2. Executive summary (counts, model versions, TEE status)
3. Article 12 compliance checklist (one checkbox per sub-obligation)
4. Decision log sample (25 records with per-record verify links)
5. Technical integrity proof (Merkle root + how to verify)
6. Certification block (CCO signature + deployer declaration)
```

## Retention and Delivery

- Reports are generated as PDF/A-3 (ISO 19005-3) for long-term archival.
- The PDF hash is anchored on-chain immediately after generation.
- Reports are stored in the enterprise's own S3-compatible store; Ochain
  holds only the hash.
- Delivery to regulators: secure download link (signed URL, 30-day expiry)
  or direct email with the PDF attached and the hash in the email body.
- Regulators with direct API access may use `GET /v1/audits/{id}/download`.
